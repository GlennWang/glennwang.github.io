HTML的全局屬性中，class跟id有什麼差別?

  1. 重複使用：

    • class： 可以在一個HTML頁面中多次使用相同的class。一個元素可以同時擁有多個class。
    • id： 在整個HTML頁面中，每個id值必須是唯一的。一個元素只能有一個id。

  2. CSS選擇器：

    • class： 在CSS中，你可以使用.（點）來選擇一個class。例如，.my-class。
    • id： 在CSS中，你可以使用#（井號）來選擇一個id。例如，#my-id。

  3. JavaScript操作：

    • class： 你可以使用JavaScript操作具有相同class的多個元素。
    • id： 由於id必須是唯一的，使用JavaScript時，通常更容易選擇和操作單個元素。

  4. 樣式優先順序：

    • 如果一個元素同時擁有相同屬性的class和id，id的樣式優先於class的樣式。這是因為id的特殊性較高。
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <style>
    /* 使用 class */
    .box {
      width: 100px;
      height: 100px;
      background-color: lightblue;
      margin: 10px;
      padding: 10px;
    }

    /* 使用 id */
    #special-box {
      background-color: lightcoral;
    }
  </style>
  <title>Class 與 ID 範例</title>
</head>
<body>
  <!-- 使用 class 的元素 -->
  <div class="box">Box 1</div>
  <div class="box">Box 2</div>
  <div class="box">Box 3</div>

  <!-- 使用 id 的元素 -->
  <div id="special-box" class="box">Special Box</div>
</body>
</html>

在這個例子中：

  • 使用了class名為 "box" 的樣式來定義一個基本的方框樣式。
  • 使用id名為 "special-box" 的樣式，覆蓋了一個具有相同 "box" class的特殊方框，使其背景色為lightcoral。
  • "box" class被應用到多個元素上，而 "special-box" id僅被應用到一個元素上。

也就是說，在定義重複樣式的時候首先考慮的是class，如果有進一步客製化需求時，則使用id。

在HTML中H1是不是最好只有一個，H1~H6各有什麼用途?

HTML中的標題標籤<h1>到<h6>用於定義文檔的標題結構，其各自的用途和應用如下：

  1. <h1>：最高級標題，表示頁面的主題或主要部分的標題。在一個頁面中，應該只有一個<h1>，用於描述整個頁面的主題。

  2. <h2>：次級標題，用於表示頁面主題的子部分或更詳細的內容。可以有多個<h2>，表示頁面的不同章節或主題的不同方面。

  3. <h3>：用於表示更具體的子標題或章節中的子部分。可以在<h2>的下面使用多個<h3>。

  4. <h4>、<h5>、<h6>：分別用於表示較低級別的標題，用於更詳細的子部分。<h4>比<h3>更具體，依此類推。在頁面中，可以根據需要使用多個<h4>、<h5>和<h6>。

在使用標題標籤時應考慮頁面的邏輯結構，確保標題的巢狀關係和級別符合文檔的層次結構。正確使用標題有助於提高文檔的可讀性、可訪問性和搜索引擎優化。

為什麼有時候position: absolute;看起來是失效的?

當我們使用 CSS 中的 position 屬性時，我們可以控制元素的定位方式。其中，relative 和 absolute 是兩種不同的定位方式。

  1. position: relative;:

    • 元素的位置相對於其原始位置。換句話說，元素會保持在文檔流中的原始位置，但你可以使用 top、right、bottom、left 屬性來調整元素相對於原始位置的偏移量。

    • 如果某個元素設置為 position: relative;，那麼它的子元素（如果有的話）的定位，通常是相對於這個父元素。

    • 例子：

      .parent {
    position: relative;
}

.child {
    position: absolute;
    top: 10px;
    left: 20px;
}

  2. position: absolute;:

    • 元素的位置相對於其最近的非static（非默認）定位的父元素。如果沒有這樣的父元素，則相對於文檔的 <html> 元素。

    • 這意味著給定元素不再保持在文檔流中，而是可以自由移動，通常是相對於最接近的非static父元素。

    • 例子：

      .parent {
    position: relative;
}

.child {
    position: absolute;
    top: 10px;
    left: 20px;
}

簡單來說，如果你發現position: absolute;失效，嘗試在你預想中的父容器裡面加一個position: absolute;。

AJAX跟Fetch API差在哪?

AJAX（Asynchronous JavaScript and XML）和Fetch API都是用於在瀏覽器中發送HTTP請求的技術，但它們之間有一些重要的區別：

  1. 介面設計:

    • AJAX: 基於XMLHttpRequest物件，它是瀏覽器提供的原始API之一。AJAX的使用通常需要設定回調函數，以處理成功或失敗的請求。
    • Fetch API: 提供了更現代且更強大的接口，使用Promise概念，更容易理解和使用。

  2. 語法:

    • AJAX: 使用XMLHttpRequest對象，包括設置回調函數、監聽事件等。具有比較複雜的語法。
    • Fetch API: 使用fetch函數，它是一個簡單而強大的API，提供了更簡潔的語法，並返回Promise對象。

  3. 預設行為:

    • AJAX: 在預設情況下，AJAX並不處理JSON格式的數據，需要額外的處理。它也不支援流式操作。
    • Fetch API: 在處理JSON數據方面更為方便，並支援流式操作。

  4. 錯誤處理:

    • AJAX: 通常使用回調函數進行錯誤處理。
    • Fetch API: 使用Promise的catch方法進行錯誤處理，使錯誤處理更一致且更容易理解。

總的來說，Fetch API提供了更簡單、現代且強大的方式來處理HTTP請求，特別適用於當前JavaScript的標準。它更符合Promise的使用，並具有更清晰的語法，使開發者更容易理解和使用。

要怎麼讓圖片按照一定比例縮放?

.img-container {
    position: relative;
    width: 100%; /* 設置父元素的寬度 */
    padding-bottom: 50%; /* 2:1 比例的高度，為寬度的一半 */
}

img {
    position: absolute;
    top: 0;
    left: 0;
    width: 100%;
    height: 100%;
}

在 HTML 中使用：

<div class="img-container">
    <img src="your-image.jpg" alt="Your Image">
</div>

img 會根據 .img-container 的寬度來動態調整，同時保持 2:1 的高度和寬度比例。
